# 투자 매뉴얼 v4 (판정 기준)

> 최종 수정: 2026-07-01 · **v4.1**: **종목상한 우선**(2·3=15%, 4=10%) · 단계 합계는 **권장 범위**만. (v4.0 15/45/40 고정 폐지 · ⑧ 폐지 · ③④⑤ 유지)

## 4단계 + 비중 규율

### 원칙 — 종목상한(강제) > 단계합(권장)

| 구분 | 성격 | 위반 시 |
|---|---|---|
| **종목당 상한** | **강제** | 초과분 트림(§A.10) |
| **단계별 합계** | **권장 범위** | 분기말 점검·메모. **강제 리밸 금지** |
| **종목 수 상한** | **강제** | 승급 보류 또는 동단계 하향(규칙 6) |

### 종목당 비중 상한 (강제)

| 단계 | 종목당 상한 | 종목 수 상한 | 역할 |
|---|---|---|---|
| **1단계** | **0%** | 무제한 | 관망. G1/G2 미통과 |
| **2단계** | **15%** | **3** | **적립** — G3s≤1. 급여 1순위 |
| **3단계** | **15%** | **3** | **유지** — G3s=2·G4=1. 추매 금지(급여 2순위 예외) |
| **4단계** | **10%** | **4** | **수확** — ride·트림. 추매·급여 **금지** |

> NBIX 예외: 비-AI 분산 슬롯 **20%** (POSITION_CAP, 단계상한과 별도).
> 첫 편입은 **2단계**부터. equal weight **강제 없음** — 종목상한 내에서 conviction·급여로 채움.
> **하이브리드 표기 금지** (`2-3`, `3-4`, `4-A` 단독 사용 금지). 4-A/4-B는 4단계 **서브타입**으로만 기록.

### 단계별 합계 — 권장 범위 (참고)

| 단계 | 권장 합계 | 현 포트 예시(8종) |
|---|---|---|
| **2단계** | **10~25%** | RKLB 15% |
| **3단계** | **30~50%** | AAOI·CLS·RMBS 45% |
| **4단계** | **25~45%** | ALAB·CRDO·NBIS·BE 40% |

> 범위는 **종목 수·상한에서 자연히 나오는 결과**에 가깝게 잡음. 2단계 0종이어도 3·4 합계를 억지로 맞추지 않음.
> 구 55/30/15·15/45/40 고정·공석 모드(§A.7b) **전부 폐지**.

### 4단계 서브타입
- **4-A (ride):** 추정치 분기 상향 지속 → 트림 없음. 레버리지·추가매수 금지.
- **4-B (트림):** 촉매 1개 → 목표비중의 1/3 기계적 트림 → **2단계(⑥·⑦)** 회전.

## 구성 제한
- **종목당 상한** — 2·3단계 **15%** / 4단계 **10%** (위 표. **강제**)
- **단계별 종목 수** — 2단계 **3** / 3단계 **3** / 4단계 **4**
- **POSITION_CAP 20%** — NBIX 등 예외 슬롯만. 일반 종목은 단계상한이 먼저
- **연결·광학 체인 상한 30%** — AAOI+CRDO+ALAB 합산 (옵션 B)
- **그로스 익스포저 상한 115%** (레버리지 사용 시. 미사용 100%)

### 배분 원칙 (v4.1)
- **equal weight 강제 없음.** 각 종목은 단계상한까지 허용, 합계는 권장 범위로 **사후 점검**
- 신규 편입·급여: 해당 종목 **종목상한 미달**일 때만. 4단계는 예외 없음
- 2단계 2종 이상일 때 종목당 **5% 미만**이면 파이프라인 1종만(MIN_EDGE 10%p) — 과도 분산 방지
- 단계 합계가 권장 **상한** 초과 → G3s 최저 종목부터 **종목상한** 기준 트림 검토 (단계합 맞추기 위한 일괄 리밸 **금지**)

## 단계 판정 — 결정트리 (v3.1 · 3·4단계 배타적)

> **폐지:** G1~G4 합산 점수로 단계 결정 — G3 모멘텀과 무관해 3·4가 겹침.
> **원칙:** 아래 트리를 **위에서부터** 첫 매칭만 적용. "3 또는 4" 표기 금지.
> **판정 주기:** G3·G3s는 종목별 실적 후 갱신, **단계·G4·G1/G2·리밸은 분기 기준일 일괄**(§A.10). 킬라인만 상시.

### 입력 변수 (각 0/1 + G3s)

| 변수 | 정의 |
|---|---|
| **G1** | 수요 충족 (업종별 치환표). 히스테리시스 적용(§A.5) |
| **G2** | 수익성 충족 (업종별 치환표). 히스테리시스 적용(§A.5) |
| **G3** | 직전 분기 컨센 상향 여부 (0/1). 상세 §A.1 |
| **G3s** | **G3 모멘텀 레벨** (0·1·2·3·4, **상한4**). 감쇠형: G3=1→+1, G3=0→−1(하한0). **「연속 분기 수」아님** — §A.2 |
| **G4** | 가격 과열 여부 (0/1). **G4=0=과열(고점 부근), G4=1=여유(충분히 조정됨)**. 상세 §A.3 |

**G4 계산 (수정 — v3.1 역전 오류 정정):**

```
drawdown = (52주 고점 − 현재가) / 52주 고점
G4=0 (과열): drawdown ≤ 20%  — 고점 대비 20% 이내 = 아직 비쌈
G4=1 (여유): drawdown > 20%  — 20% 넘게 빠짐 = 3단계·2단계 유지에 유리
```

### 결정트리 (순서 고정)

```
① 킬라인·투자 이유반증 발화?        → 즉시 이탈 (단계 무관)
② G1=0 OR G2=0?               → 1단계(관찰)
③ G3s ≥ 3 AND G1=1 AND G2=1?  → 4단계 · 서브타입 4-A (ride)
④ G3s ≥ 2 AND G4=0 AND G1=1 AND G2=1? → 4단계 · 서브타입 4-B (과열·수확)
⑤ G3s = 2 AND G4=1 AND G1=1 AND G2=1? → 3단계  ← 3단계는 여기만
⑥ G3s ≤ 1 AND G4=1 AND G1=1 AND G2=1? → 2단계 (적립·급여 대상)
⑦ G3s ≤ 1 AND G4=0 AND G1=1 AND G2=1? → 2단계 (과열·추가매수 금지)
```

> **⑧ 폐지 (v4):** G3s=0이라도 G1=1·G2=1이면 **⑥ 또는 ⑦** (2단계 적립). 1단계는 **② G1/G2 미통과**만.
> **강등:** 보유 종목 G3=0 → §A.4로 G3s−1 후 트리 재적용. G3s=0·G1✓·G2✓ → 2단계 유지(적립), 1단계 아님.

### 3단계 vs 4단계 — 한 줄 구분

| 단계 | 유일한 조건 |
|---|---|
| **3단계** | G3s **= 정확히 2** + G4=1 + G1·G2 통과 |
| **4단계 4-A** | G3s **≥ 3** + G1·G2 통과 (가격 무관·ride). **③이 ④보다 우선** |
| **4단계 4-B** | G3s **≥ 2** + **G4=0** + G1·G2 통과. **G3s≥3이면 ③으로 4-A** |

> **겹침 없음:** G4=0이면 3단계 불가(⑤ 미충족). G3s≥3이면 무조건 4-A(③).

### 단계별 의미 (역할)

| 단계 | 펀더멘털 상태 | 포트 행동 |
|---|---|---|
| 1 | G1/G2 미통과 (투자 이유 미확인) | 비중 0% |
| 2 | 투자 이유 확인·G3s≤1 (모멘텀 미성숙) | 급여 1순위·신규 편입 |
| 3 | G3s=2(모멘텀 L2)·G4=1 | 유지·추매 금지 |

| 4-B | G3s≥2·G4=0 (과열) | 트림·2단계 회전 |

## 승급·강등 (G3s 기준)

| 이동 | 조건 |
|---|---|
| 1→2 | G1·G2 통과 + 결정트리 ⑥ 또는 ⑦ + 킬 0 |
| 2→3 | G3s 1→2 (+1 갱신) + G4=1 → ⑤ |
| 3→2 | G3=0 연속 등으로 G3s 2→1 → ⑥⑦ (적립 복귀) |
| **2→4-B** | G3s 1→2 **동시** G4=1→0 → 결정트리 ④ (과열·수확 직행). G3s≥3이면 ③→4-A |
| 3→4-A | G3s 2→3 (+1 갱신) |
| 3→4-B | G3s≥2 유지 + G4 1→0. **G3s≥3이면 4-A(③)** |
| **모멘텀 하락** | G3=0 → G3s −1 → **§A.4** (0으로 리셋 **금지**) |
| 즉시 이탈 | 킬라인·투자 이유 틀림 (§A.6) |

## 판정 출력 포맷 (고정)

```
[TICKER] stage=3 subtype=- G3s=2 G1✓ G2✓ G4✓ triggered_rule=⑤ action=유지
[ALAB] stage=4 subtype=4-A G3s=4 G1✓ G2✓ G4✗ triggered_rule=③ action=ride·추매금지
```

- `subtype`: 4단계만 `4-A` 또는 `4-B`. 그 외 `-`
- **`triggered_rule`**: 매칭된 결정트리 번호(①~⑧) **필수**
- **G3s 숫자 필수** — 3·4 논쟁 시 G3s만 보면 됨
- G4 표기: ✓=G4=1(여유), ✗=G4=0(과열)

## 종목별 킬신호

> **원칙 (§A.6):** "추정치 상향 멈춤" **단독** = G3=0(강등 1칸)이지 킬 아님. 킬은 **추가조건 AND** 필수.

### 2단계
- **AAOI:** 800G 양산 지연 2분기+ / 주 고객(MSFT·AWS 등) 이탈 / GM 30% 이하 하락 지속
- **NBIX:** 핵심 제품(Ingrezza) 매출 역성장 / FDA CRL·상한요법 실패 / EPS 추정치 2분기 연속 하향
- **RMBS:** 로열티 매출 2분기 역성장 / 주요 고객 DDR 인소싱 발표 / 메모리 IP 경쟁 대체
- **CLS:** AI·데이터센터 매출 비중 감소 / OP margin 2분기 연속 하락 / 단일 고객 40%+ 집중 악화

### 3단계
- **CRDO:** ~~추정치 상향 멈춤~~ → G3=0만. **킬:** 하이퍼스케일러 인소싱 **또는** PAM4 경쟁자 대형 수주 탈취
- **RKLB:** 분기 매출 역성장 2분기 / Neutron 발사 실패 2회+ / Neutron 2027 추가 연기

### 4단계
- **ALAB:** Customer A ≥35% + B/C/D 절대매출 동시 감소 / ~~추정치 상향 멈춤~~(→G3=0) / capex 조기경보 2개+ (§A.6b). **킬:** Customer A 조건 **또는** 조기경보 2개+ **동시**
- **NBIS:** ARR 3분기 연속 감속 / 추론 매출 기여 0 / 희석 가속
- **BE:** GPM 악화 2분기 / DC 계약 이행 차질 / CEO 증자 필요 발언

### 마스터스위치 연쇄 (자금줄별 — 일괄 "AI capex" 점검 금지)

하이퍼스케일러 2027 capex 가이던스 하향 시 **해당 자금줄만** 점검:

| 연쇄 강도 | 자금줄 | 대상 종목 |
|---|---|---|
| **직격** | 광학모듈·연결IC·서버제조·GPU클라우드 | AAOI·CRDO·ALAB·CLS·NBIS |
| **후행 1~2분기** | 메모리IP | RMBS |
| **약연쇄** | DC전력 | BE (전력 병목은 별 예산·별 타이밍) |

**마스터스위치** 조기경보 2개+ = **직격 4줄만** 리스크오프. 메모리IP·DC전력·독립줄은 별도 판정. (ALAB 종목 킬용 조기경보 목록은 §A.6b — 별도 정의)

## 리밸런싱 규칙
1. **급여·신규 자금** — **1순위:** 2단계(⑥·G4=1·킬 OFF·**종목상한 15% 미달**). **2순위:** 3단계(⑤·G4=1·킬 OFF·**종목상한 15% 미달**). **4단계** 신규·급여 **금지**
2. 3단계: 유지. 추매 금지 (규칙 1의 2순위만)
3. 4단계: 4-B 트림 현금 → **2단계(⑥·⑦)** 회전
4. 킬신호 → 전량/대폭 청산 → watchlist 1순위 대체 (MIN_EDGE 10%p)
5. capex 마스터스위치 하향 → **직격 4줄** 종목만 우선 트림. 독립줄(NBIX·RKLB)은 연쇄 매도 금지
6. **슬롯 초과:** **§A.10 기준일**에만. 종목 수 상한 초과 시 승급 보류 또는 G3s 최저 하향
7. **Drift (v4.1):** **§A.10 기준일**에만. **종목상한 초과** 시 초과분 트림(강제). 단계 합계가 권장 범위 밖이면 **점검·메모만**(강제 리밸 금지). POSITION_CAP 20% 초과도 트림

## 레버리지 프로토콜 (v3.2)
- **허용:** 2단계 종목만, G4=1, 킬 OFF, **pre-profit 태그(§매핑표) 종목 합산 비중 <50%**
- **상한:** 그로스 105% (기본) / 115% (공격 모드 — 운영자 1줄 기록 시에만)
- **금지:** 3·4단계, G4=0(과열), 킬 ON, 4-A ride 구간
- **강제 청산:** 그로스 >115% 또는 VIX>30 5일 연속 또는 킬 1건 이상 → 레버 0%

## 금지
- 하이브리드 단계 표기 (`2-3` 등)
- 1단계(관찰)에 포트 비중 배정
- 4단계·과열(G4=0)에 레버리지
- "수주"와 "자본조달" 혼동 (승급 근거 기록 시 범주 명시)
- G3s를 **「연속 분기 수」**로 해석 (반드시 §A.2 모멘텀 레벨)
- G3=0 시 G3s **0으로 일괄 리셋** (반드시 §A.4 −1칸만)
- 분기 중 G4 변동만으로 3↔4-B 전환 (킬라인 제외)

## AI 판정 가이드라인 (v4.0 부록)

LLM·사람 판정 시 본문 트리보다 **아래를 우선** 적용. 모호 시 §번호를 출력 `triggered_rule`에 기록.

### A.1 G3 정의 (컨센서스)
- **소스:** FactSet 컨센서스 **단일 고정** (Bloomberg 등 교차검증은 참고만)
- **비교 시점:** 직전 분기 실적 발표 후 **3거래일** 종가 시점 NTM 컨센 vs 그 직전 스냅샷(이전 분기 판정 시점)
- **G3=1:** NTM EPS **AND** NTM Revenue **둘 다** ≥+1.0% 상향 (소수 반올림 0.1%p)
- **G3=0:** 하나라도 하향, 또는 둘 다 +1% 미만(정체 포함)
- **보조:** 실적 당일 가이던스 상향은 G3=1 **가산 증거** (단독으로 G3=1 불가)
- **신규 종목(워치리스트):** 직전 스냅샷 없으면 **G3 판정 보류**. FactSet NTM 스냅샷 **2개 확보 후** 첫 G3·G3s 시드(보통 G3=1이면 G3s=1로 시작). 스냅샷 1개만 있으면 G3s=0·G1/G2 통과 시 **⑥(2단계)**.

### A.2 G3s = G3 모멘텀 레벨 (감쇠형) — 「연속」용어 금지
- **정의:** G3s는 **추정치 모멘텀 점수**. "G3=1이 연속인 분기 수"가 **아님**.
- **상한:** `G3s ≤ 4` (L4=4-A ride 상한). G3=1로 5+ 누적 **금지** — 4-A 좀비 방지.
- **갱신 (종목별 실적 후 3거래일, §A.10):**
  - G3=1 → `G3s = min(4, G3s + 1)`
  - G3=0 → `G3s = max(0, G3s − 1)` (§A.4)
- **의미:** G3=0 한 분기 후에도 G3s>0일 수 있음(감쇠 잔존). 4-A G3s=4 → G3=0 → G3s=3 → ③ 유지는 **감쇠 설계**. 상한 4이므로 최대 감쇠 거리=4칸(≈1년) 이내 강등권 진입.
- **트리 임계:** G3s≤1→2단계권, =2→3단계, ≥3→4-A권, ≥2·G4=0→4-B권. **≥5 없음**(상한으로 4가 최대).
- **AI 출력:** "연속 N분기" 표현 **금지**. `G3s=N (모멘텀 레벨, 상한4)`만 사용.

### A.3 G4 확정·히스테리시스
- **확정 시점:** 분기 판정일(§A.2) **또는** 월말 종가 중 **늦은 날 1회만**
- **가격:** 5거래일 종가 평균 vs 52주 고점으로 drawdown 계산
- **히스테리시스:** G4=0 진입 drawdown ≤18% / G4=1 진입 drawdown ≥22% (띠 4%p)
- **분기 중** 일별 G4 변동으로 3↔4-B 전환 **금지** (① 킬라인만 상시)

### A.4 모멘텀 갱신 (G3=0 시 −1칸)

```
1) G3s ← max(0, G3s − 1)   ※ 0으로 리셋 금지 · "연속 깨짐→0" 해석 금지
2) 갱신된 G3s·G1·G2·G4로 결정트리 ①~⑧ 재적용
3) triggered_rule에 최종 매칭 번호 기록
```

**예:** 4-A G3s=4 → G3=0 → G3s=3(모멘텀 잔존) → ③ 4-A 유지. 3단계 G3s=2 → G3=0 → G3s=1 → ⑥⑦ → 2단계.

### A.5 G1·G2 히스테리시스 (깜빡임 방지)
**G1 (book-to-bill 등):**
- **편입(G1=0→1):** book-to-bill ≥ 1.00 (2분기 연속 권장)
- **이탈(G1=1→0):** book-to-bill < 0.95
- 0.95~1.00 구간: **직전 분기 G1 유지**

**G2 (수익성 % 지표 — 동일 띠 규칙):**
- **편입(G2=0→1):** 임계 **이상** (예: OPM ≥5%, GM ≥70%, NRR ≥105%)
- **이탈(G2=1→0):** 임계 **−1%p 미만** (예: OPM <4%, GM <69%, NRR <104%)
- **띠 구간**(임계~임계−1%p): **직전 분기 G2 유지** — 4.9%↔5.1% 왔다갔다 해도 ② 추락 깜빡 없음

| 업종 G2 | 편입(=1) | 이탈(=0) | 띠(유지) |
|---|---|---|---|
| EMS OPM | ≥5.0% | <4.0% | 4.0~5.0% |
| 메모리IP GM | ≥70% | <69% | 69~70% |
| SaaS NRR | ≥105% | <104% | 104~105% |
| 기타 % 지표 | 임계 이상 | 임계−1%p 미만 | 그 사이 유지 |

### A.6 킬 vs G3=0 분리
| 사건 | 처리 |
|---|---|
| 추정치 상향 멈춤(정체) | **G3=0만** → §A.4 강등 |
| 추정치 하향 2분기 | G3=0 + 해당 종목 킬 후보 검토 |

### A.7 급여·종목상한 (v4.1)
- 급여는 **종목상한**(2·3=15%, 4=10%) 미달 종목에만. 단계 합계 권장 범위는 급여 배분 **기준 아님**
- 2단계 1순위 → 3단계 2순위. 2단계 0종이어도 **단계합 맞추기 위한 억지 매수 금지**
- 4단계는 종목상한·급여 **둘 다** 금지
- 단계 합계가 권장 **하한** 미만 → 급여 여유. **상한** 초과 → 신규 매수 중단·트림 검토(종목상한 위반부터)

### A.8 4-A ride 가드 (선택)
- 4-A 유지 중 52주 고점 대비 drawdown **>35%** (5일 평균) → **4-B 전환 검토** (자동 킬 아님)
- 킬 미발화 시 ride 유지 가능. 운영자 메모 1줄 권장

### A.10 분기 기준일·이벤트 갱신 (9종목 비동기)
9종목 실적일이 분산(1~3월 등)되므로 **두 축**으로 분리:

| 축 | 시점 | 갱신 대상 | 포트 행동 |
|---|---|---|---|
| **이벤트** | 각 종목 실적 후 **3거래일** | G3 → G3s (§A.2) | **기록만**. 단계 변경·리밸 **실행 금지** |
| **기준일** | 달력 분기 **마지막 거래일** 1개 (Q1=3월말·Q2=6월말·Q3=9월말·Q4=12월말) | G4·G1·G2 확정, 결정트리→단계, Drift·슬롯초과·리밸 | **일괄 실행** |

- **기준일 전** 해당 종목 실적 미발표 → 그 종목 G3/G3s는 **직전 확정값 유지**(신규는 §A.1 보류).
- **예외:** ① 킬라인·투자 이유반증은 **상시** 즉시 (기준일 무관).
- **AI 출력:** `event_date=YYYY-MM-DD`(실적+3TD)와 `rebalance_date=YYYY-MM-DD`(분기말) **둘 다** 기록. "오늘 실적 떴으니 지금 리밸?" → **아니오**, 기준일까지 대기.
